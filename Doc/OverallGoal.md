第 1 步：重构核心数据结构（枚举与常量）  
原项目中有大量关于音符、八度、键盘状态的基础定义。在开始复杂逻辑前，先提取这些“静态字典”。  
重构目标：剥离旧代码中的平台特定宏定义（如 Windows），将音频配置、按键状态等统一转换为强类型的现代 C++ 枚举（如使用 juce::Enumeration 和 juce::EnumValue）。  
预期结果：生成一个独立、无平台依赖的 Constants.h 文件，将所有静态数据（如音符、八度、键盘状态）集中管理，并为后续模块提供强类型数据支持。  
第 2 步：重构按键映射逻辑（Key Mapping）  
FreePiano 的灵魂在于它复杂的按键布局。我们需要提取“按键按下 -> MIDI 音符”的计算逻辑，抛弃它原有的存储机制。  
重构目标：将电脑键盘的扫描码（Scan code）映射转化为标准的 juce::MidiMessage。  
预期结果：获得一个独立的 KeyMapper 类。此时程序依然没有声音，但内部已经具备了将键盘输入转化为 MIDI 数据流的能力。  
第 3 步：重构音频与 VST 宿主层（Plugin Host）  
这是技术难点。旧版可能直接调用了原生的 VST SDK，我们需要用 JUCE 的 AudioPluginFormatManager 替代它。  
重构目标：使用 JUCE 的 AudioPluginFormatManager 和 AudioDeviceManager 实现跨平台音频流处理和 VST 插件加载。  
预期结果：生成一个封装好的 PluginHost 对象。它能够挂载一个 VST3 乐器，并能够处理音频块（Audio Block）。  
第 4 步：串联事件循环，让程序“发声”  
现在我们要把前面的模块组装进主程序。  
重构目标：打通数据流：键盘事件 -> Mapper -> MIDI 数据 -> VST 插件 -> AudioDeviceManager 音频输出。  
预期结果：运行程序，按下电脑键盘的 A、S、D、F 等按键时，如果成功加载了 VST 插件，你应该能听到钢琴声了！这是重构最重要的里程碑。  
第 5 步：重构 UI (虚拟钢琴键盘)  
抛弃旧的 GDI 绘图，由于 JUCE 自带钢琴 UI 组件，这一步非常轻松。  
重构目标：提供现代化的、不卡顿的矢量交互界面。  
预期结果：窗口中出现一排精美的虚拟钢琴按键，支持鼠标点击发声，也支持随电脑键盘的高亮联动。  
第 6 步：高阶功能恢复（布局切换与录制导出）  
最后处理那些繁杂的非核心逻辑。  
重构目标：使用 JUCE 现代的状态管理工具替换陈旧的序列化代码，增加程序的健壮性。  
预期结果：用户可以在 UI 上保存当前键盘映射为 Preset，或者加载其他的布局文件。

注意：不要引入 windows.h，我们使用的是跨平台的 JUCE 框架，请使用 JUCE 的等效类。
