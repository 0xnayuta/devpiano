# ADR 0002: 旧 FreePiano 源码只作为迁移参考

## 状态

已采用。

## 背景

仓库中保留了旧版 FreePiano 源码 `freepiano-src/`。这些代码对于理解历史功能、默认布局、消息语义和旧模块职责很有价值。

但旧项目包含大量平台绑定实现，例如原生 Windows GUI / GDI、WASAPI / ASIO / DirectSound 后端、旧式 VST SDK 风格宿主逻辑，以及旧配置与序列化方式。直接复制这些实现会把旧架构问题带入新的 JUCE 重构版。

## 决策

`freepiano-src/` 只作为迁移参考资料，不作为当前实现的一部分。

具体原则：

- 可以阅读旧代码以理解历史行为、业务规则、默认布局和功能边界。
- 不直接复制旧的 Windows 平台实现到新代码中。
- 不重新引入以 `windows.h` 为中心的依赖链作为核心架构。
- 不原样沿用旧 VST SDK 风格接口、GDI GUI 或原生音频后端。
- 新实现优先使用 JUCE 抽象重建。
- 迁移重点是“提炼行为”，不是“直接复刻实现”。

## 影响

正面影响：

- 新代码保持现代、跨平台、可维护。
- 降低旧项目平台绑定和全局状态设计对新架构的污染。
- 允许在理解旧行为的同时，用 JUCE 重新设计模块边界。

代价：

- 一些旧功能无法直接快速复制，需要重新建模和验证。
- 迁移时需要先理解旧行为，再设计新实现。
- 部分高级功能，例如录制、回放、导出，需要先定义现代事件模型。
